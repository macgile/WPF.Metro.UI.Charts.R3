.NET – Les graphiques [Partie 1 : utilisation générale]

2 avril 201621 octobre 2016 Loïc

Dans cet article nous verrons comment tracer des graphiques en .NET. Par contre, nous ne verrons pas comment en créer à partir de zéro : j’aime bien réinventer la roue, mais d’autres le font déjà. Je présenterai donc ici une bibliothèque (ou plutôt deux) gratuite qui vous permettra de faire des graphiques facilement. Ces graphiques sont plutôt jolis (ils se veulent correspondre à une interface type « Metro ») et animés (clic, chargement, modification de valeur…).

 

Tout d’abord, une petite précision : pour ceux qui sont intéressés par les graphiques en Silverlight ou pour les mobiles, cela est possible. Dans cet article nous verrons comment faire des graphiques en WPF, car la bibliothèque présentée n’est faite que pour ça, mais elle se base sur une autre bibliothèque qui est compatible avec ces technologies :

  • WPF
  • Silverlight
  • Windows 8

 

Cette partie abordera rapidement les bibliothèques et comment les utiliser. Nous ferons un point plus approfondi sur la customisation dans un autre article.

 

Sans plus attendre, plongeons-nous dans le vif du sujet !

 

La bibliothèque utilisée est WPF Metro UI Charts. Comme indiqué précédemment, cette bibliothèque est une amélioration de Modern UI (Metro) Charts for Windows 8, WPF, Silverlight, car elle contient des graphiques supplémentaires. Tout d’abord, téléchargez la DLL de la bibliothèque ici, puis ajoutez-la en tant que référence dans votre projet. Il s’agit de la bibliothèque améliorée que j’ai très légèrement modifiée pour y corriger un bug (cf. les nuages de point).

Pour faciliter les explications (et le code en général) j’utiliserai ici l’architecture MVVM. A vrai dire, j’utilise cette architecture dans tous mes projets WPF. Cela permet de bien séparer le code, et on peut la mettre en place facilement à l’aide de la bibliothèque MVVM Light.

 

Il faut savoir que pour afficher des valeurs contenues dans une liste, il est nécessaire de passer par des objets complexes. Cela est dû au fait que les graphiques utilisent les propriétés DisplayMember et ValueMember pour chaque élément de la liste, qui correspondent respectivement à la propriété contenant le nom de la valeur (string) et à la propriété contenant la valeur (int…).

Nous avons donc besoin d’une classe contenant au moins ceci :

{
public string Libelle { get; set; }
 
public int Valeur { get; set; }
}

new ChartItem { Libelle = "Item 4", Valeur = 13},

Voici enfin la liste des graphiques disponibles avec à chaque fois le code XAML correspondant. Vous pourrez voir que le code est très similaire pour presque tous ces graphiques, ceux différents étant ceux rajoutés à la bibliothèque originale.

 

 

I – Les graphiques de base

Voici les graphiques inclus dans la bibliothèque initiale. On peut voir quelques attributs indispensables :

  • ChartTitle et ChartSubTitle : le titre et le sous-titre du graphique, qu’on peut masquer avec la propriété ChartTitleVisibility. Notez cependant qu’il n’est pas possible de masquer uniquement le sous-titre.
  • SeriesTitle : le titre de la série, doit être unique pour chaque graphique sur le même tableau. Non affiché si il y a un seul graphique.
  • ItemsSource : la source des données à afficher
  • DisplayMember : la propriété de l’objet à afficher qui correspond au libellé de la légende (type string)
  • ValueMember : la propriété de l’objet à afficher qui correspond à la valeur (type int, double…)
  • SelectedItem : la source de la donnée sélectionnée

II – Les graphiques supplémentaires (GAMultipleTypeSeriesChart)

Tous ces graphiques ont été ajoutés à la bibliothèque de base, et ces ajouts ne sont pas anodins. Leur construction est légèrement différente : ils sont tous de type GAMultipleTypeSeriesChart, et contiennent des séries qui elles sont de type différent.

La seule propriété différente à noter est DataPointType. C’est cette propriété qui déterminera le type de graphique à afficher.

 

Il est cependant à noter que ces graphiques ne fonctionnent que si un style est présent pour chaque diagramme : télécharger le style de base. Une fois extrait, vous pouvez inclure le fichier XAML dans votre projet et l’ajouter en tant que ressource dans le fichier App.xaml :

1
2
3
4
5
6
7
8
<Application.Resources>
<ResourceDictionary>
<vm:ViewModelLocator x:Key="Locator" d:IsDataSource="True" xmlns:vm="clr-namespace:ApplicationTestWpf.ViewModel" />
<ResourceDictionary.MergedDictionaries>
<ResourceDictionary Source="Graphs.xaml"></ResourceDictionary>
</ResourceDictionary.MergedDictionaries>
</ResourceDictionary>
</Application.Resources>
